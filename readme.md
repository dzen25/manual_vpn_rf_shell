5. Оплачиваем, ждём письмо с IP-адресом, логином `root` и паролем

### Подключение к серверу

После покупки нужно подключиться к серверу по SSH.

**Windows:**
- Скачать [PuTTY](https://www.putty.org/) или использовать встроенный терминал Windows (cmd / PowerShell)
- В терминале набрать:
```bash
ssh root@<IP-адрес-сервера>
```
- Ввести пароль из письма (символы при вводе не отображаются — это нормально)

**macOS / Linux:**
- Открыть терминал, набрать:
```bash
ssh root@<IP-адрес-сервера>
```

После подключения вы увидите командную строку сервера — можно начинать.

## 2. Настройка файрвола (защита сервера)

Файрвол — это фильтр, который блокирует все подключения к серверу, кроме разрешённых. Без него сервер будет открыт для всех.

Сначала разрешаем нужные порты:

```bash
# Разрешаем SSH (удалённый доступ) — без этого потеряете доступ к серверу!
ufw allow ssh

# Разрешаем порт 443 — через него будет работать VPN
ufw allow 443/tcp
```

Теперь нужно выбрать порт для панели управления. Это должно быть случайное число от 10000 до 60000. Например, 39282. Запомните его — он понадобится дальше.

```bash
# Замените 39282 на свой выбранный порт
ufw allow 39282/tcp
```

Включаем файрвол:

```bash
ufw enable
```

Система спросит подтверждение — набираем `y` и нажимаем Enter. Проверяем:

```bash
ufw status
```

Должно быть видно три разрешённых порта: 22 (SSH), 443 и ваш порт панели.

## 4. Защита панели

Прямо сейчас ваша панель доступна по прямому адресу. Боты и сканеры могут её найти. Чтобы этого не произошло — скрываем её за секретным путём.

В панели:

1. Заходим в **Настройки**
2. Находим **Путь панели (URL Root Path)**
3. Вписываем любой путь, например: `/mypanel/`
4. Сохраняем

Теперь панель доступна только по адресу:
```
http://<IP>:<порт>/mypanel/
```

Старый адрес без пути перестанет работать — это и есть защита.

## 5. Создание VPN-подключения (VLESS Reality)

VLESS Reality — самый современный протокол. Трафик маскируется под обычное HTTPS-соединение с крупным сайтом (Google, Microsoft). Блокировки его не видят.

В панели:

1. Заходим в **Подключения** (в левом меню)
2. Нажимаем **Добавить подключение**
3. Заполняем:

| Параметр | Значение | Комментарий |
|----------|---------|-------------|
| Протокол | `vless` | Не менять |
| Порт | `443` | Стандартный HTTPS-порт |
| Transmission | `tcp` | |
| Security | `reality` | Обязательно reality, не TLS |
| uTLS | `chrome` | Маскировка под браузер Chrome |
| Dest (SNI) | `dl.google.com:443` | Сайт, под который маскируемся |
| Server Names | `dl.google.com` | Тот же домен без порта |
| Private / Public Key | Нажать кнопку генерации | Сгенерируются автоматически |
| Short ID | Нажать кнопку генерации | Или оставить пустым |

4. Нажимаем **Добавить клиента** внизу формы
5. Вписываем имя (например, `phone` или `laptop`)
6. Нажимаем **Создать**

### Получаем ссылку для подключения

После создания в списке подключений появится новая строка. Нажимаем на иконку (QR-код или копирование) рядом с именем клиента — получаем ссылку вида:

```
vless://xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx@IP:443?...
```

Эту ссылку нужно добавить в VPN-клиент на телефоне или компьютере (см. раздел 8).

## 6. Установка Cloudflare WARP (обход детекта VPN)

### Зачем это нужно

Google, YouTube, Gemini и другие сервисы умеют определять, что ваш IP принадлежит дата-центру, а не домашнему провайдеру. В результате — ограничения, капчи, блокировки.

WARP — бесплатный сервис от Cloudflare. Он подменяет выходной IP сервера на «чистый» адрес из сети Cloudflare, который не числится в базах как датацентр.

### 6.1. Установка

Выполняем команды по порядку:

```bash
curl -fsSL https://pkg.cloudflareclient.com/pubkey.gpg \
  | gpg --dearmor -o /usr/share/keyrings/cloudflare-warp-archive-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/cloudflare-warp-archive-keyring.gpg] https://pkg.cloudflareclient.com/ $(lsb_release -cs) main" \
  > /etc/apt/sources.list.d/cloudflare-client.list

apt update && apt install -y cloudflare-warp
```

### 6.2. Настройка

```bash
# Регистрируем устройство в Cloudflare
warp-cli registration new

# Включаем режим прокси — WARP не трогает весь трафик, работает только как локальный прокси
warp-cli mode proxy
warp-cli proxy port 40000

# Подключаемся
warp-cli connect
```

### 6.3. Проверка

```bash
curl -x socks5://127.0.0.1:40000 https://www.cloudflare.com/cdn-cgi/trace
```

В выводе ищем две строки:
- `warp=on` — WARP активен
- `ip=104.28.xxx.xxx` — IP от Cloudflare (не ваш серверный)

Если видите `warp=on` — всё работает. Если нет — проверяем статус: `warp-cli status`.

### 6.4. Автозапуск

WARP автоматически стартует после перезагрузки сервера. Убедиться:

```bash
systemctl is-enabled warp-svc
```

Если ответ `enabled` — всё в порядке. Если нет:

```bash
systemctl enable warp-svc
```

### Команды управления

| Команда | Что делает |
|---------|-----------|
| `warp-cli status` | Текущий статус |
| `warp-cli connect` | Подключить |
| `warp-cli disconnect` | Отключить |

## 7. Подключение WARP к панели (маршрутизация через Cloudflare)

Сейчас WARP работает на сервере, но Xray (движок VPN) о нём не знает. Нужно сказать Xray: «трафик к Google отправляй через WARP».

### 7.1. Добавляем исходящее подключение

В панели: **Настройки Xray** → вкладка **Исходящие подключения** → **+ Создать исходящее подключение**

| Поле | Значение |
|------|---------|
| Тег | `warp-socks` |
| Протокол | `socks` |
| Адрес | `127.0.0.1` |
| Порт | `40000` |

Нажимаем **Сохранить** (кнопка вверху страницы).

### 7.2. Создаём правило маршрутизации

В панели: **Настройки Xray** → вкладка **Маршрутизация** → **+ Создать правило**

| Поле | Значение |
|------|---------|
| Domain | `geosite:google` |
| Outbound Tag | `warp-socks` |

Все остальные поля оставляем пустыми.

**Создать правило** → **Сохранить** → **Перезапуск Xray** (красная кнопка вверху).

### Результат

Весь трафик к сервисам Google (включая YouTube, Gemini, Gmail) теперь идёт через Cloudflare WARP. Остальной трафик идёт напрямую через сервер.

## 10. Решение проблем

### VPN подключается, но сайты не открываются
Проверить что Xray работает: `x-ui status`. Если нет — `x-ui restart`.

### Gemini/YouTube снова детектят VPN
WARP мог отключиться. Проверяем: `warp-cli status`. Если статус Disconnected — `warp-cli connect`, затем `x-ui restart`.

### Не заходит в панель
Проверить порт: `ufw status`. Если порта нет — `ufw allow <порт>/tcp`. Проверить панель: `x-ui status`.

### Забыли логин и пароль
```bash
x-ui settings    # показать текущие
x-ui reset       # сбросить на дефолт
```

### WARP не ставится
Убедиться что Ubuntu 22.04 или 24.04. На других версиях репозиторий Cloudflare может не работать.

### Встроенный WARP в 3X-UI (через кнопку WARP в панели) не работает
Это нормально — на многих VPS встроенный WireGuard-outbound падает с таймаутом. Рабочий вариант — `warp-cli` в режиме SOCKS5-прокси, как описано в этом гайде.

## 12. Обход VPN для российских сервисов (настройка клиента)

### Зачем

Многие российские сервисы (Госуслуги, банки, Яндекс, VK, Ozon и др.) блокируют или ограничивают доступ с зарубежных IP. Если весь трафик идёт через VPN-сервер в Амстердаме — эти сервисы могут работать некорректно или вообще не пускать.

Решение — настроить VPN-клиент так, чтобы трафик к российским ресурсам шёл **напрямую через провайдера**, минуя VPN-туннель. Остальной трафик по-прежнему идёт через VPN.

```
Трафик к google.com  → через VPN-туннель → сервер → интернет
Трафик к gosuslugi.ru → напрямую, мимо VPN → интернет
```

### Hiddify

1. Открыть приложение
2. Зайти в **Настройки** → **Config Options** → **Routing**
3. В разделе маршрутизации выставить:
   - **Bypass Domestic** (обход внутренних) → если есть такой переключатель, включить
   - Или в **Remote Domain Strategy** выбрать режим, который пропускает локальный трафик
4. Если есть раздел **Custom Rules** / **Direct Domains**:
   - Добавить: `geosite:category-ru`
5. Если есть **Direct IPs**:
   - Добавить: `geoip:ru`

### v2rayN (Windows)

1. Открыть v2rayN
2. **Настройки** → **Routing Setting** → **Advanced Function**
3. Добавить правило:

```json
{
  "outboundTag": "direct",
  "domain": ["geosite:category-ru"]
}
```

И правило по IP:

```json
{
  "outboundTag": "direct",
  "ip": ["geoip:ru"]
}
```

4. Убедиться что правило `direct` стоит выше правил прокси для совпадающих доменов

### v2rayNG (Android)

1. Открыть v2rayNG
2. Нажать три точки → **Settings** → **Routing**
3. Выбрать **Bypass mainland** или **Custom**
4. В кастомных правилах добавить:
   - **Direct Domain**: `geosite:category-ru`
   - **Direct IP**: `geoip:ru`
